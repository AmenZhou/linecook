00 / The Idea

Think of it like a kitchen

You don't need to know the internals. This whole system is just a way to hand off a task and walk away — the same way you'd clip an order ticket to the rail in a restaurant and let the kitchen take it from there.

The Analogy · A Restaurant Kitchen
📝
You write an order ticket "Make me a fix for the rate-limit bug."
📌
Clip it to the rail and leave No standing around waiting by the kitchen.
👨‍🍳
A cook grabs the next ticket Works it from start to finish, on their own.
🔔
The dish comes out, logged You review it whenever you're back.
What's Actually Happening
⌁
You write a task file A plain text note describing what you want.
📂
Drop it in the inbox/ folder A watched folder on your laptop.
◈
Claude picks it up and works The AI runs the whole task by itself.
✓
The result is saved to history With a full log of what it did.
No servers, no dashboards, no cloud — just text files and your own laptop.

03 / How to Harness

Using the
inbox flow

1
Drop a task file into inbox/ Any .md file with frontmatter fields task: and mode:
2
launchd fires drain-inbox.sh every 60s Picks up new files, registers them in .orchestrate/tasks/
3
Harness runs the orchestrate skill Executes phases autonomously — research, strategy, implementation, QA
4
Results archived to orchestrate-history/ Full run log, phases, lessons learned, and task diff preserved
Example task file

07 / Key Terms

The jargon, decoded

Every technical word in this deck, in plain language.

inbox/ A plain folder on the laptop. Drop a task file in here and it gets queued up — nothing else required.
launchd macOS's built-in scheduler — like an alarm clock for programs. Here it checks the inbox every 60 seconds, even after a reboot.
the harness The autonomous runner that lets the AI carry out an entire task on its own — no back-and-forth chat needed.
orchestrate skill The recipe the AI follows for every job: research → plan → build → check.
frontmatter The little header at the top of a task file — the task: and mode: lines that tell the system what to do.
registry The system's to-do list — tracks each task's state: pending, in progress, or done.
mode: auto / gated Auto = fully hands-off. Gated = pauses for your approval before anything risky.
tend loop The recurring check that picks up waiting tasks and nudges them along until they finish.
